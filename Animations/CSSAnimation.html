Vad är CSS-animationer?

CSS-animationer används för att skapa rörelse. och förändringar i egenskaper hos element. Med dessa animationer kan man exempelvis ändra färg, storlek eller position på element.

Det finns två huvudsakliga metoder för att animera med CSS: transitions och animations med @keyframes. De fungerar på olika sätt och används i olika situationer. Nedan följer förklaringar till båda koncepten med exempel.

CSS Transitions

En CSS transition är som en mjuk övergång mellan två tillstånd då de låter dig ändra egenskapsvärde smidigt under en given tidsperiod. Transitions kräver en utlösare, oftast en tillståndsändring som :hover.

Håll muspekaren över elementet nedan för att se en fin CSS-övergångseffekt:

CSS

Förklaring med hjälp av exemplet ovan

Först defineras det startillstånd elementet ska ha och sedan vilka CSS-egenskaper som du vill lägga till en övergång till inklusive dess varaktighet. I själva utlösaren defineras de slutvärden elementet ska ha.

                
    #demo-box {
        height: 80px;
        width: 80px;
        background-color: red;

        transition: 
            transform 1s,
            background-color 1s;
    }

    #demo-box:hover {
        transform: rotate(180deg);
        background-color: blue;
    }

Först defineras de startillstånd elementet ska ha, det vill säga hur elementet beter sig utan en utlösare. I den andra delen av #demo-box används egenskapen transition för att bestämma när övergångseffektern kommer att starta, i detta fall då bakgrundsfärgen eller rotationsvinkeln förändras.

#demo-box:hover fungerar som en utlösare, det vill säga när muspekaren hålls över elementet kommer de nya egenskapern att gälla för elementet. Då kommer övergången från den övre biten kod utföras. Observera att när markören flyttas ut ur elementet, kommer den elementet gradvis återgå till sin ursprungliga stil.

CSS Animations & @keyframes

En animation låter ett element gradvis ändras från en stil till en annan. För att använda CSS-animering måste du ange några nyckelbilder för animationen. Nyckelbilder innehåller vilka stilar elementet kommer att ha vid vissa tidpunkter.

CSS

Förklaring med hjälp av exemplet ovan

                
    @keyframes roteraOchFärg {
        0% {
            left: 0;
            transform: rotate(0deg);
            background-color: brown;
        }
        50% {
            background-color: orange;
        }
        100% {
            left: calc(100% - 100px);
            transform: rotate(360deg);
            background-color: darkgreen;
        }
    }

    #demo-circle {
        height: 80px;
        width: 80px;
        background-color: brown;
        animation: roteraOchFärg 5s infinite;
    }

Med @keyframes kan du definera flera steg i en animation, till exempel från 0 % till 100 % och bestämma hur elementet ska förändras över tid. I detta fall styr left, transform och background-color hur elementet ska bete sig. Med egenskapen animation binder man animationen roteraOchFärg till #demo-circle. Animationen tar fem sekunder och fortsätter oändligt med hjälp av nyckelordet infinite.

Transition vs. Animation

Även om transitions och animations båda skapar rörelse, används de i olika situationer.

TransitionAnimation
Kan ej loopasInga problem med looping
Kräver en utlösareHar precis börjat, behöver ingen triggerkälla
Animerar ett objekt från en punkt till en annanLåter dig definera nyckelbilder som varierar från ett tillstånd till ett annat med olika egenskaper
Passar bra för knappar och menyerAnvänds för mer komplexa rörelser
